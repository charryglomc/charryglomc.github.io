引用

  • 本质：引用是已定义变量的别名，但实际上引用变量占据存储空间，存储一个地址，编译器阻止对它本身的任何访问，从而保证引用“从一而终”
  • 声明引用时必须初始化，否则会编译出错
  • 引用一旦定义，就不能更改引用的指向

引用作为函数参数

引用作为函数的返回值

  • 函数返回变量的生命周期一定要大于函数的生命周期，一般是返回*this，static静态变量，全局变量，输入输出流

  • 不能返回局部变量的引用

  • 不能返回new创建的堆空间变量的引用：new创建的堆空间变量是一种临时变量，如果返回的话，在内外都delete不了，会造成内存泄漏。

内联函数

  • 在c中我们使用带参数的宏定义优化时间
  • 在c++中我们则使用inline内联函数优化时间
  • 使用宏定义容易出错，也不可以调试，且不能操作类的私有数据成员
  • inline函数可以调试
  • 不宜使用inline函数的两种情况：
    • 函数体内代码比较长
    • 函数体内出现循环或其他复杂的控制结构
  • 往往选择短小简单的函数内联

栈和堆区别

  • 栈由编译器自动管理，不要我们手工管理；而堆需要我们手工管理，且易造成内存泄漏
  • 栈只能访问一个栈帧范围的的内存地址；但是堆跟内存大小有关，可以访问的地址很大。
  • 栈一般是静态分配，如局部变量；堆一般是动态分配，如new
  • 对于堆来说，生长方向是向上的，也就是向着地址增大的方向生长；对于栈来说，生长方向是向下的，也就是向着地址减小的方向生长

派生类对象和基类对象之间的相互转换

  • 派生类对象可以转换为基类对象(向上转型是安全的)，即派生类对象利用赋值运算符函数赋给基类对象（记：16字节可以赋给8字节）

  • 基类的指针可以指向一个派生类对象 (记：8字节的指针可以指向一块16字节的内存空间)

  • 基类的引用可以绑定到一个派生类对象

  • 派生类对象不可以转换为基类对象，即派生类对象不能利用赋值运算符函数赋给基类对象，但是可以用static_cast强制向下转型（不安全）

  • 派生类的指针不可以指向一个基类对象 (记：16字节的指针不可以指向一块8字节的内存空间，不安全)，可以用static_cast强制向下转型（不安全）

  • 派生类的引用不可以绑定到一个基类对象（不安全）

面向对象的四大特征：抽象，封装，继承，多态

  • 多态的概念：对于同一指令信号（比如警车鸣笛），针对不同的对象（普通行人，警察，罪犯）产生不一样的行为（避让，抓捕，逃跑）

  • 多态激活的5个基本条件：

    1、基类要定义虚函数

    2、派生类要重定义（重写）该虚函数

    3、创建派生类对象

    4、用基类的指针指向（引用绑定到）派生类的对象

    5、基类的指针（引用）去调用虚函数

  • 多态类型：

    • 静态多态：函数重载，运算符重载，模板，发生在编译阶段（静态联编，早期联编）
    • 动态多态：通过虚函数体现，发生在运行阶段 （动态多态，晚期联编）
    • 动态多态的体现必须要有虚函数，但调用虚函数不一定能体现动态多态

  • 虚函数：成员函数前面加virtual

  • 纯虚函数：virtual void display() = 0; virtual void display() = 0;

如何通过虚函数体现多态？

  • 基类定义自己的虚函数，派生类重写该虚函数，在基类对象的存储布局上多存储一个虚函数指针，该指针指向基类的虚函数表（虚表），该虚表存放的是基类虚函数的入口地址；在派生类对象的存储布局上也多存储一个虚函数指针，该指针指向派生类的虚函数表（虚表），该虚表存放的是重写了的虚函数的入口地址。

哪些函数不能被设计为虚函数?

普通函数：是一个非成员函数，所以不能定义为虚函数

静态成员函数：发生时机在编译时候，虚函数体现多态在运行时；静态成员函数没有this

内联成员函数：发生时机在编译时候，虚函数体现多态在运行时；内联成员函数前面加上virtual关键字，编译器不会报错，但是此时已经失去内联的函数

友元函数：如果友元函数本身是一个非成员函数（普通函数）的时候，是不能被设计成虚函数的；但是如果友元函数本身是一个成员函数的时候，是可以设计成虚函数

构造函数：发生时机在编译时候，虚函数体现多态在运行时；如果构造函数是虚函数，虚函数的调用会满足虚函数调用的逻辑结构，会入虚表，会有虚函数指针，而构造函数的作用是为了初始化数据成员的，此时构造函数没有调用完成，就证明对象还没有构建完成，那虚函数指针就不一定存在，就调不到虚表，所以构造函数不能被设计为虚函数

抽象类：一个类有至少一个纯虚函数

声明了纯虚函数的类称为抽象类，抽象类不能创建对象。

如果派生类继承自抽象类，但是派生类没有将抽象类中的所有纯虚函数进行实现的话，该派生类也是一个抽象类，该抽象类也不能创建对象。

如果一个类的构造函数被protected进行修饰，那么该类就是抽象类，抽象类不能创建对象。

虚析构函数

当基类的析构函数被设计虚函数的时候，派生的析构函数会被自动变成虚函数。此时对于编译器而言，就是一个虚函数的重写。对于程序员而言，属于一个名字不一样的重写。编译器会将析构函数解释为destructor,所以编译就认为析构函数就是重写了。析构函数一个类只有一个，既有唯一性。
