运算符重载

友元

形式：friend 表达式

友元的三种形式

  • 以普通函数的形式
  • 以成员函数的形式
  • 以友元类的形式

友元也是可以重载的

友元的性质

  • 友元是单向的，不具备传递性，不能被继承
  • 由于友元破坏了封装性，所以需要慎用

运算符重载

为什么需要运算符重载

  • 为了让自定义类型的数据的操作与C++内置的数据类型的数据的操作形式一致

五种运算符不能重载

  • 1、成员访问符 .
  • 2、成员指针运算符 .*
  • 3、域运算符 ::
  • 4、条件运算符 ?:
  • 5、长度运算符 sizeof

规则六条

  • 1、重载的运算符的操作至少有一个是自定义类型或枚举类型，即：两个内置类型不能
  • 2、优先级与结合性不变
  • 3、不会改变运算规则，操作数个数，顺序
  • 4、重载运算符不能有默认参数
  • 5、重载逻辑运算符(&& ||)后不再具备短路求值特性
  • 6、不能臆造不存在的运算符

形式

  • 以普通函数形式

    • 需要get函数支撑

  • 以友元函数的形式

  • 以成员函数的形式

总之：与我们的C++预定义的运算符保持一致

普通运算符重载

  • 双目运算符:+ - * / %

    • 建议友元

  • 复合赋值+= -=

    • 建议成员函数，对象本身发生了变化

特殊运算符重载

  • 自增自减

    • 前置++与后置++的区别(包括重载形式的区别)

      • 效率
      • 取地址

  • 重载小括号(函数调用运算符)

    • 与普通函数的区别

      • 函数执行完毕之后生命周期就结束了，但是对象没有
      • 函数对象可以携带状态但是普通函数没有
      • 即使函数定义静态变量，携带状态，但是没有封装性
      • 函数对象的产生就是为了减少函数指针的使用

    • 与函数指针的区别

      • 定义方式不一样，使用形式一样
      • 函数对象可以携带附加数据，但是函数指针不行

  • 重载中括号(下标访问运算符)

    • const版本

    • 两个操作数：左操作数是对象本身，右操作数是任意类型

    • 增加C++中数组检索的安全性

    • 返回值是引用，可以作为左值

      • 左值与右值的区别？适时复习，右值我们讲了哪些种

  • 输入输出流

    • 输出流

      • 相当于实现print方法

    • 输入流

      • 注意程序的健壮性，以及与之前学习的流类比

        • 流的四种状态

  • 箭头解引用

    • 箭头运算符的返回值必须是一个指针或者重载了箭头运算符的对象
    • 解引用，返回对象引用
    • 举例：三层类Data，MiddleLayer(数据成员是Data类型指针)，ThirdLayer(数据成员是MiddleLayer类型指针)
    • 好处：可以一步到位，调用最上层成员函数

类型转换

  • 其他类型向自定义类型进行转换

    • 其他类型可以是内置类型与自定义类型

      • 内置内类型

        • 隐式转换

          • 通过构造函数完成

      • 另一种自定义类型

        • 构造函数
        • 赋值运算符函数

  • 自定义类型向其他类型转换

    • 类型转换函数

      • 形式：operator 类型(),例如：operator int(){}
      • 要求：必须是成员函数；参数列表中没有参数；没有返回值，但在函数体内必须以 return 语句返回一个目标类型的变量(对象)；一般情况下，不推荐使用，违反常规思维
