类和对象

1、面向对象的思想、定义格式、class与struct比较

面向对象思想：三兄弟住店的例子,从面向过程与面向对象两个方面去理解，对象通过彼此之间的交互，状态发生改变

  • 类
    数据成员(属性)
    成员函数(操作/行为)

定义形式

  • class 类名{public: protected: private: };

类的实现

  • 类的实现就是类的成员函数的实现

类的使用

  • 通过类创建对象，然后调用类的成员函数

封装性的体现

  • 在类外不能访问类的private数据成员与成员函数

    • 引申出类的内部

class默认权限是私有的，struct默认权限是公有的，struct在C++中是做了扩展的

代码规范的讲解

  • public，private修饰区域，在代码中写的地方
  • 成员函数、成员变量的写法
  • 重要性

在类外部定义成员函数

  • 形式：加类名与作用域限定符

头文件与实现文件分开，放在头文件与实现文件中进行编译

  • 1、工程比较大，不可能就一个文件
  • 2、在类内部实现的成员函数是inline函数

2、对象的创建与销毁

对象的创建：构造函数

  • 形式：函数名与类名相同，没有返回值，但是可以传参数，所以构造函数可以重载的
  • 作用：完成数据成员的初始化
  • 默认情况下，系统会提供一个默认(无参)的构造函数

初始化表达式

  • 真正初始化数据成员的地方
  • 数据成员的初始化顺序，只与数据声明的顺序有关，与其在初始化表达式中的顺序无关

对象的销毁：析构函数

  • 形式：函数名与类名相同，并带一个取反符号；没有返回值，没有参数，具有唯一性，不支持函数重载

  • 作用：完成对象的销毁，执行清理任务，如：Computer堆内存的释放

  • 默认情况，系统会自动提供析构函数，但是可能没有作用

  • 使用：可以显示调用，但是不建议显示使用，可能造成释放后还在使用，转到Computer例子，继续使用打印函数看打印结果

    • Computer带两个参数构造函数的参数解释_brand

  • 析构函数的调用时机，全局对象、栈对象、静态对象、堆对象

3、拷贝构造函数

形式：类名(const 类名 &rhs)，例如:Point(const Point &rhs)

默认情况下编译器会自动提供一个

  • 浅拷贝

    • Point例子，只有基本内置类型

  • 深拷贝

    • Computer例子，内含指针

调用时机三种

  • 1、一个已经存在对象复制给一个新对象

  • 2、行参和实参都是对象，形参与实参结合

  • 3、函数返回值是对象，函数调用完成返回

    • 编译器优化g++ Point3.cpp -fno-elide-constructors

提问：拷贝构造函数的引用与const可以去掉吗？

  • 如果去掉引用符号，形参是对象，会无穷递归调用拷贝构造函数

  • const不能去掉，当传递右值(临时对象)，无法调用拷贝构造函数

    • 右值

      • 不能取地址

        • 临时对象
        • 匿名对象
        • 字面值常量

    • 左值

      • 可以取地址

4、this指针

本质：指向对象本身

位置：隐藏在于每个非静态的成员函数的第一个参数处

形式：指针本身(指向)不能修改，是：指针常量 *const

使用方式

  • 例子:pt.print(&pt)或者Point::print(&pt)

5、赋值运算符函数

形式：类名 & operator=(const 类名 &rhs)，例如：Point &operator=(const Point &rhs)

默认情况下，编译器会提供一个赋值运算符函数

返回值类型

  • 返回值必须为引用，否则会符合拷贝构造函数的调用时机，增大开销，且执行完毕后左操作数是存在的

  • 返回值不能为void

    • 连等pt1 = pt2 = pt3

完整的实现形式

  • “四部曲”

    • 自复制
    • 释放左操作数
    • 深拷贝(包含空间的申请)
    • 返回*this

  • 注意左操作数与右操作数的提醒

6、特殊数据成员的初始化

const成员

  • 只能在初始化表达式中进行，而且不能在构造函数等任何成员函数内部赋值

引用reference成员

  • 只能在初始化表达式中进行初始化
  • 引用成员占据一个指针空间大小

类对象成员

  • 自定义类型创建的对象作为另一个类的对象成员时，必须显示初始化

static成员

  • 必须在类外进行初始化，被类的所有对象所共享，如果有头文件与实现文件，需要时实现文件进行定义

  • 不占类的空间，sizeof测试

    • 换成double
    • 内存对齐，专题讲解

7、特殊成员函数

static成员函数

  • 静态成员函数没有隐含的this指针
  • 只能调用静态数据成员与静态成员函数
  • 如果非要在静态成员函数中访问非静态成员，可以传参
  • 除了使用对象进行调用，还可以使用类名进行调用静态成员函数

const成员函数

  • const成员函数的this指针类型 const 类名 * const this
  • 与非const成员函数可以进行重载，且const成员函数不能修改数据成员
  • 非const对象可以调用两种，但是默认调用非const成员函数
  • const对象只能调用const成员函数

8、对象组织

const对象

  • 只能调用const成员函数

指向对象指针

对象数组

堆对象

  • 单个堆对象
  • 堆对象数组

9、单例模式

要求：通过该类只能创建一个对象

  • 不能是栈对象
  • 只能是堆对象，且在类内部

应用场景

  • 只要出现全局对象的地方都可以使用单例模式

  • 全局唯一的资源

    • 配置文件、网页库、字典库、日志系统写的日志对象
